# 《启动大纲》— 精品餐厅官网

> 基于三个 Dribbble 参考设计稿综合梳理：Drizzle Restaurant Website Design / Restaurant Landing Page UI / Restaurant Website UI Responsive Design

---

## 一、品牌与项目核心

- **项目名称 / 品牌名**：Drizzle（暂定，取"淋酱、细流"之意，暗示精致与流动感）
- **一句话定位**：一家注重食材、空间与仪式感的精品餐厅官网，让用餐前就先"吃"到氛围
- **核心价值主张**：在正式踏入餐厅之前，让每一位访问者通过屏幕感受到厨房的温度、食材的新鲜和空间的质感——"看着就想订位"
- **目标受众**：
  - 来旅游的用餐客户（通过搜索引擎/社交媒体发现，对"打卡"友好的视觉有期待）
  - 本地美食探索者（25–45 岁，注重用餐体验胜过价格，会在小红书上分享）
  - 商务宴请需求方（在手机上快速判断"请客是否有面子"）
- **品牌个性关键词**：温暖、精致、自信、有机、沉浸

## 二、设计方向锁定

从下列美学方向中选择一个，或者描述你自己的混合风格。选定即执行，不妥协。

- ~~残雕主义 Brutalism / Degraded~~
- **▶ 编辑风 Editorial** ← 主要基调（大图大字的杂志感排版，让食物成为叙事主角）
- ~~暗黑奢华 Dark Luxury~~
- **▶ 有机自然 Organic / Biophilic** ← 辅助调性（暖奶油底 + 自然材质纹理 + 食材本身的色彩作为点缀）
- ~~瑞士极简 Swiss Minimalism~~
- ~~复古现代 Retro Modern~~
- **自定义混合描述**：**编辑风 × 有机自然**——大图大字的杂志叙事结构 + 暖奶油底色的呼吸感 + 锈红/琥珀色作为品牌的"热信号"。不冷、不暗、不过度设计，让食物和空间自己说话。

**视觉情绪参考：**

- [Drizzle Restaurant Website Design](https://dribbble.com/shots/26722436-Drizzle-Restaurant-Website-Design) — 完整网站设计，关注页面间叙事流
- [Restaurant Landing Page UI Design](https://dribbble.com/shots/26881394-Restaurant-Landing-Page-UI-Design) — 落地页聚焦，关注第一屏的信息密度与 CTA 策略
- [Restaurant Website UI Responsive Design](https://dribbble.com/shots/27456134-Restaurant-Website-UI-Responsive-Design) — 响应式设计，关注移动端菜单网格、药丸导航、印章徽章、Koulen 超粗体标题

**必须避开的风格陷阱：**

- 不要科技蓝渐变（破坏食物温暖的氛围）
- 不要通用快餐红黄配（麦当劳式的廉价感）
- 不要弥散阴影 / glassmorphism（毛玻璃效果与"有机"调性冲突）
- 不要手写花体字（降低可读性，与编辑风的精准感矛盾）
- 不要自动轮播 Hero Slider（打断用户主动浏览节奏）

## 四、字体系统（非 Inter/Roboto）

| 角色 | 字体名称 | 来源 | 字重使用范围 |
|---|---|---|---|
| **显示字体 Display** | **Koulen** | Google Fonts | 400（仅此一字重，本身就是 compressed bold） |
| **正文字体 Body** | **DM Sans** | Google Fonts | 400 Regular, 500 Medium, 700 Bold |
| **中文 fallback** | **Noto Serif SC** | Google Fonts | 400, 700（用于页面中可能的中文展示） |
| **等宽/特殊** | **JetBrains Mono** | Google Fonts | 400（仅用于价格数字、厨房计时器等场景） |

**排版性格：**

- **Koulen 标题**：全大写 / 宽字距（`letter-spacing: 0.08em`）/ 行高 0.95–1.05，制造压缩大字的杂志封面感。单页只用 1–2 次，不滥用。
- **DM Sans 正文**：行高 1.6–1.75，营造呼吸感。段落最大宽度 65ch。
- **数字与价格**：使用 DM Sans 的 tabular-nums 特性，对齐数字排版。
- **禁止**：标题不使用斜体；正文不用全大写；不对 Koulen 加任何字重变换（没有粗体版本）。

## 五、影像与素材策略

### 定制摄影/素材计划

| 类别 | 规格 | 内容描述 | 色彩方向 |
|---|---|---|---|
| **Hero 主视觉** | 16:9 横版，桌面 2560×1440，移动 750×1334 | 餐厅招牌菜的极近特写：酱汁淋下的瞬间 / 刀切入食材的截面 / 蒸腾的热气 | 暖黄光，浅景深，高饱和食材本色 |
| **空间摄影** | 3:2 横版 | 餐厅内景：黄昏时的靠窗座位、开放式厨房的火焰、蜡烛映在酒杯上的光斑 | 低色温，暗部有细节，不过曝 |
| **食材特写** | 1:1 正方形 | 应季食材的排列：新鲜香草、手工意面、熟成牛肉、应季蔬菜 | 自然侧光，木/石/麻布纹理为背景 |
| **人物/氛围** | 4:3 横版 | 厨师在备餐的手部特写 / 客人的自然笑容 / 侍酒师倒酒的瞬间 | 不摆拍，捕捉"被偷拍"的自然感 |
| **菜单菜品** | 4:5 竖版 | 每道菜的俯拍或 45° 角，统一光源方向和背景 | 浅景深，菜品居中，背景虚化 |
| **纹理素材** | — | 亚麻布、手工纸、陶器表面、木纹、石墙 | 作为背景纹理层，低透明度叠加 |

### 图标系统倾向

- **风格**：线性图标（Line Icons），描边粗细 1.5px
- **端点**：圆头（round cap），连接处圆角（round join）
- **尺寸体系**：16px（行内） / 24px（按钮内） / 32px（功能图标） / 48px（装饰图标）
- **色彩**：默认使用 `#6B5E53`（次文本色），交互态使用 `#C75B39`（强调色）
- **推荐图标集**：Phosphor Icons（风格统一、线性为主、覆盖全面）

### 避免使用的素材库/风格

- ❌ Unsplash 的"微笑吃沙拉"商务图（虚假感毁掉食欲）
- ❌ AI 生成的食物图片（手指数量不对、酱汁流向违反物理）
- ❌ 过度饱和的 HDR 食物照（看起来像塑料模型）
- ❌ 通用扁平插画风格的食材/厨师（与编辑风调性冲突）

## 六、页面结构与内容层级

| 页面 | 核心目的 | 关键内容块 |
|---|---|---|
| **首页** | 3 秒建立"想来"的欲望 → 推动订位 | ① Hero 大图 + 品牌标题（Koulen） + 主 CTA（订位） ② 概念文案（2–3 句，编辑风排版） ③ 招牌菜品横向滚动卡片（3–5 道） ④ 空间氛围图 + 短引文 ⑤ 主厨/团队一句话 ⑥ 再次订位 CTA ⑦ Footer（地址/时间/社交/邮件订阅） |
| **菜单** | 展示菜品、激发食欲、支持筛选 | ① 分类导航 pill（前菜/主菜/甜品/酒水） ② 菜品网格（图片 + 菜名 + 简介 + 价格） ③ 食材标签（素食/无麸质/主厨推荐徽章） ④ 酒单独立区块 |
| **关于/理念** | 建立信任、讲述故事 | ① 主厨大幅肖像 + 理念引文 ② 食材溯源故事（编辑风长文排版 + 穿插图片） ③ 空间设计哲学 ④ 时间线（品牌历程） |
| **活动/私宴** | 展示包场/活动的可能性 | ① 私宴空间展示 ② 活动类型卡片（生日/商务/婚礼） ③ 咨询表单 |
| **联系/订位** | 完成转化 | ① 订位表单（日期/时间/人数/特殊需求） ② 联系方式（电话/邮箱/微信） ③ 地图嵌入（位置） ④ 营业时间 ⑤ FAQ 折叠面板 |
| **博客/日记**（可选） | SEO 长尾内容 | ① 文章卡片列表（食材科普/主厨手记/活动回顾） ② 分类/标签筛选 |

**全局叙事流：**

访问者从首页进入→ Hero 大图 + Koulen 标题形成"冲击力第一印象"→ 向下滚动，招牌菜品卡片触发"食欲"→ 空间氛围图建立"想来"的情绪→ 主厨引文建立"信任"→ CTA 推动"订位"。整个流程 === **一见倾心 → 越看越饿 → 相信品质 → 立刻订位**。

## 七、动效与交互调性

### 整体动效哲学

- ~~锐利精准型~~（150–250ms，ease-out）
- **▶ 舒缓呼吸型**（400–600ms，ease-in-out）← 餐厅需要的不是"快"，而是"舒适"
- ~~极简无感型~~

### 动效权重偏好（1 最重要）

1. **微交互触感** — 按钮的磁吸按压、卡片的微妙响应、导航 pill 的切换触感
2. **滚动联动** — 视差慢移、滚动揭示、图片随滚动轻微缩放
3. **叙事入场** — 文字逐行错峰出现、菜单卡片从下淡入
4. **环境氛围** — 极慢速的暖色渐变呼吸、背景纹理微动
5. **页面过渡** — 页面切换的柔和叠化（不抢内容）

### 必须存在的动效

- ▶ 按钮的磁吸按压感（hover 微升 2px + 阴影扩散 / tap 缩回至 98%）
- ▶ 卡片悬停的微妙抬升（hover 上浮 4px + 阴影加深，200ms ease-out）
- ▶ 文字逐行错峰入场（每行延迟 80–120ms，从下向上 20px + opacity）
- ▶ 图片的视差慢移（滚动速度 × 0.85，仅对非 Hero 的背景装饰图）
- ▶ 页面切换的叠化过渡（300ms opacity 交叉淡入淡出）
- ▶ 骨架屏加载（菜单页的菜品卡片，用暖色块 + shimmer）
- ▶ 数字递增动画（"已服务 10,000+ 食客" 触发递增计数）
- ✘ 自定义光标样式（保持原生，不增加认知负担）

### 必须避免的动画

- ▶ 弹跳（bounce）效果
- ▶ 无限循环的旋转/呼吸灯
- ▶ 视差滚动超过 2 层
- ▶ AOS 库的默认淡入（改用 GSAP ScrollTrigger 精细控制）
- ▶ 全页 loading spinner（用骨架屏替代）
- ▶ 元素持续抖动/摇摆
- ▶ Hero 区的自动轮播幻灯片

### 对标网站动效参考

1. **URL — Drizzle Restaurant Website Design** — 欣赏的动效细节：观察页面切换的品牌过渡方式
2. **URL — Restaurant Landing Page UI Design** — 欣赏的动效细节：观察 Hero 区域的 CTA 入场时机和按钮反馈
3. **URL — Restaurant Website UI Responsive Design** — 欣赏的动效细节：观察菜单网格滚动加载的节奏、pill 导航的切换动画、移动端菜单的展开方式

## 八、技术与性能硬指标

- **目标加载时间**：LCP ≤ 1.8s（移动端 4G），FID ≤ 50ms
- **可访问性等级**：WCAG 2.2 AA（色彩对比度最低 4.5:1，大文本 3:1）
- **必须支持的设备/浏览器**：
  - 移动端：iOS Safari 最新 2 个大版本 / Android Chrome 最新 2 个大版本
  - 桌面端：Chrome / Firefox / Safari / Edge 最新 2 个大版本
  - 最小支持宽度：320px（iPhone SE 级别）
- **暗色模式**：跟随系统（prefers-color-scheme），同时提供手动切换入口
- **国际化需求**：中英双语（中文为主，英文为国际化兜底）
- **技术约束**：
  - 纯静态站点（SSG）优先 — 推荐 Astro 或 Next.js static export
  - CSS 方案：Tailwind CSS 4 + 自定义 Design Token
  - 动画库：GSAP（ScrollTrigger + Flip）
  - 图片格式：WebP 优先 + AVIF 兜底，使用 `<picture>` 标签响应式适配
  - 图标：Phosphor Icons（SVG sprite 或按需导入）
  - 表单：使用 Web3Forms 或 Resend API（轻量、无需自建后端）
  - 部署：Cloudflare Pages 或 Vercel（全球 CDN，免费层足够）

### 图片性能策略

```
缩略图（卡片）: 400×300 WebP, quality 75
菜品大图: 800×1000 WebP, quality 80
Hero 桌面: 2560×1440 WebP, quality 75, lazy load = eager
Hero 移动: 750×1334 WebP, quality 70, lazy load = eager
纹理/装饰: 200×200 以下, AVIF, quality 60, 低透明度
```

### 断点策略

| 断点 | 宽度 | 设计列数 | 关键变化 |
|---|---|---|---|
| Mobile | 320–767px | 4 列 | 单栏布局，汉堡菜单，Hero 标题缩小至 32px |
| Tablet | 768–1023px | 8 列 | 两栏菜单网格，底部固定 CTA 条隐藏 |
| Desktop | 1024–1439px | 12 列 | 三栏菜单网格，全导航展开，视差生效 |
| Wide | 1440px+ | 12 列（居中容器 1280px） | 最大内容宽度 1280px，两侧留白 |

## 九、灵感与禁区

### 对标网站 / 最欣赏的 3 个网站

**1 URL — 欣赏**：[Drizzle Restaurant Website Design](https://dribbble.com/shots/26722436-Drizzle-Restaurant-Website-Design)

- 欣赏的点：完整的多页面网站设计，观察页面间的叙事流、品牌一致性、菜单展示方式

**2 URL — 欣赏**：[Restaurant Landing Page UI Design](https://dribbble.com/shots/26881394-Restaurant-Landing-Page-UI-Design)

- 欣赏的点：落地页的信息架构——如何在第一屏内建立欲望 + 推动行动，Hero 区域的构图策略

**3 URL — 欣赏**：[Restaurant Website UI Responsive Design](https://dribbble.com/shots/27456134-Restaurant-Website-UI-Responsive-Design)

- 欣赏的点：移动优先的响应式策略（320px 起）、药丸形分类导航、超粗体 Koulen 标题的杂志感、传统印章徽章的品牌细节、菜单网格在桌面/移动端的自适应变化

### 绝对禁忌（可能会毁掉你品牌的事情）

- **照片是网站的灵魂**——绝对不能用 AI 生成的菜品图或通用 stock photo，必须实拍。一张假的食物照片会让整个网站的品牌信用归零
- **PDF 菜单**——不要把菜单做成 PDF 下载链接。对 SEO 不可见、移动端体验灾难、用户流失的确定性原因
- **自动播放音乐或视频音效**——餐厅网站不是夜店，突然出声会让办公室里的用户社死
- **订位按钮藏在深处**——"订位"CTA 必须在每一屏都触手可及（sticky 或每个 section 尾部重复出现）
